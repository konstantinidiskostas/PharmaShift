# 🏥 PharmaShift

> **Status: 🚧 In Active Development**
> A professional scheduling solution designed for modern pharmacies(adjustable to other professions), built with Spring Boot and React.

## 📖 Project Overview
**PharmaShift** is a Full-Stack web application currently being developed to automate and optimize the weekly employee scheduling process for a local pharmacy.

Unlike generic calendar tools, this project focuses on the **"Nurse Scheduling Problem"** applied to the pharmacy sector. It aims to replace manual planning with an intelligent system that ensures legal compliance, operational coverage, and fair shift distribution among staff.

### 🎯 The Real-World Challenge
The project is designed to solve complex scheduling constraints such as:
* Maintaining mandatory pharmacist presence at all times.
* Adhering to Labor Law requirements (e.g., the 11-hour rest rule between shifts).
* Tracking historical data to ensure "Fairness" in weekend and closing shift rotations.

---

## 🛠️ Tech Stack
* **Backend:** Java 17+, Spring Boot 3, Spring Data JPA.
* **Frontend:** React.js, JavaScript (ES6+), Tailwind CSS.
* **Database:** H2 Database (File-based for local persistence).
* **Version Control:** Git & GitHub.

## 📈 Key Features (Planned)
* **Automated Constraint Validation:** Real-time alerts for labor law violations or coverage gaps.
* **Historical Analytics:** Tracking "Saturday shifts" and "Closing shifts" per employee to ensure fair distribution.
* **Local-First Architecture:** Designed to run locally on the manager's machine without the need for complex server infrastructure.
* **Drag-and-Drop Interface:** An intuitive React-based calendar for quick manual adjustments.
